Key design decisions

Three calls worth flagging.

The dashboard is where loyalty is built or lost — open daily by anyone with an active case. These calls keep it usable on both a 5-inch Android and a 27-inch monitor.

01 / Two-shell pattern

Side nav on desktop, tab bar on mobile

Persistent 240 px sidebar with grouped links + user chip on desktop. Sticky bottom tab bar (Home / Apps / Messages / Me) on mobile. They share routes but never both render. Top bar on mobile carries only brand + avatar to free up vertical space.

02 / Welcome banner = most urgent action

One thing that matters now

The dark ink banner at the top of the dashboard surfaces the one action that needs the user (upload photo, confirm fee, etc.). Replaces the generic "welcome back" string. Empty dashboards show a "let's get started" instead.

03 / In-app status hero

Same enum, compressed

The application detail uses a shorter version of the F5 public tracker hero — same colors, same status enum, but inline buttons replace the agent-card link (because the agent card sits in the right column already).
