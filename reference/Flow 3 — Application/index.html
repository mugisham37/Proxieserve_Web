Key design decisions

Three calls worth flagging.

This is the surface where most applications die — at form-friction. Every choice below was made to keep someone with low digital literacy moving on a flaky phone over a metered 3G connection.

01 / Auto-save indicator

Status, not a save button

Users on flaky connections want to know their work is safe. The auto-save indicator in the top bar shows three states (saved / saving / offline) — the only system status that's always visible. No explicit "Save" button means one less thing to forget.

02 / Confirmation code

Big, monospace, copyable

The application code is the only thing a guest user has linking them back to their case. Display-size mono with copy + WhatsApp share buttons, repeated in SMS and WhatsApp delivery. The page caption literally says "TREAT IT LIKE A PASSWORD".

03 / Schema migration

Versioned, transparent

When admin updates a service form (Flow 9), drafts in progress could break. We version every schema and show users a "What changed?" card with the diff on resume — so trust survives the inevitable form revisions.
