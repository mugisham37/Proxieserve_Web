Password show/hide is a real <button aria-pressed>

OTP group has role="group" + aria-label; each cell has aria-label="Digit N"
Tab toggles use role="tablist" + aria-selected
Banners use role="alert" on error and aria-live="polite" on info
Strength meter colour-only feedback is paired with a text label
Focus visible on every interactive element via tokens.css :focus-visible
Reduced motion respected globally
Touch targets ≥ 44 × 44 px (cells, toggles, buttons)
Lockout timer announced via aria-live="polite" updates every minute
Resend button disabled state announced (aria-disabled + visible "Resend in 29s")
10 /

Hand-off & self-review

Engineering notes

  • Sessions. JWT in HttpOnly; Secure; SameSite=Lax cookie. Refresh token rotation on every use. 30-day idle timeout for clients; 4-hour for staff.
  • Password hashing. Argon2id with sensible parameters (m=64MB, t=3, p=4). Pepper from Vault. Minimum length 8; we reject the OWASP 10,000-most-common list.
  • 2FA. TOTP (RFC 6238, 30s window, ±1 step). SMS via Twilio + Africa's Talking failover. Backup codes are 10 single-use 8-char alphanumeric strings.
  • OTP. 6-digit numeric. 5-minute expiry. 3-attempt cap before invalidation. Constant-time comparison server-side.
  • Rate limiting. 5 failed login attempts per account per 10 min = 5 min lockout. Per-IP cap also: 30 sign-in attempts per IP per 10 min.
  • Account enumeration. Reset-password and claim-by-code endpoints respond identically regardless of whether the account/code exists.
  • Magic links. 32-byte URL-safe tokens. One-time use, 15-min expiry. Each new link invalidates the previous.
  • Device trust. 30-day device cookie tied to refresh token. New device triggers OTP regardless of TOTP. Revocable from settings (F6 / F8 / F9).
  • Role routing. Server-side role claim. Client signs in → F6. Agent → F8. Admin → F9. Insufficient role redirects to staff sign-in with return URL preserved.
  • Email verification. Persistent banner in dashboard until verified. Account works (reads); writes (new applications, payments, profile edits) blocked.
  • Analytics events. auth_signin_attempt, auth_signin_success, auth_signin_fail {reason}, auth_signup_start, auth_signup_complete, auth_otp_sent {channel}, auth_otp_verified, auth_otp_expired, auth_forgot_requested, auth_password_reset, auth_claim_attempt {found:bool}, auth_2fa_method {totp|sms|backup}, auth_lockout, auth_account_exists_hint.
