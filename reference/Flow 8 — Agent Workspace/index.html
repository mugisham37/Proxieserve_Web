Key design decisions

Three calls worth flagging.

The agent zone is where the product's promise — "someone competent will handle this for me" — actually gets delivered. These calls protect speed and prevent the costly mistakes.

01 / Checklist-first

Tell the agent what's next

Case detail leads with a per-service checklist showing exactly where the agent is ("⟵ you are here") and the remaining steps, each tied to a status. It's a guided workflow, not a bare status dropdown — onboarding a new agent is far faster.

02 / Note safety

Internal notes can't leak

Note mode turns the whole composer yellow and changes the label + button. Beyond the UI, the API filters internal notes out of any client-facing read at the query layer — so a note can't leak even through the API, not just the screen.

03 / Keyboard-first

The mouse is optional

J/K to move, ↵ to open, E to change status, R/N for reply/note, ⌘K for the command palette. A fast agent processing 40 cases a day never has to reach for the mouse — and a persistent hint bar teaches the shortcuts.
