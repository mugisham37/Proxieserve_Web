Flow 8 — Agent Workspace.

The fulfilment engine. Dense, keyboard-first, used all day under time pressure. Inherits the F6 app shell, tightens it, and adds the dual-lane conversation that keeps internal notes from ever reaching a client.

Accessibility checklist

SideNav is <nav>; active link aria-current="page"; counts in accessible text
Queue table is a real <table> with header scope; rows keyboard-focusable
All keyboard shortcuts have a visible hint bar + ? full-list dialog
Command palette: role="dialog", focus-trapped, arrow-key list, Esc closes
Status changer is a labelled <select>; change announced via live region
Internal-note mode announced ("Internal note — only agents see this") beyond colour
Conversation messages distinguish client / agent / note / system semantically, not by colour alone
Doc viewer zoom controls labelled; QC results carry icon + text
Dark mode meets the same AA contrast floor as light
Confirm dialogs (reassign, reject, flag) focus-trap + require explicit action
SLA / priority never colour-only — paired with text ("Over by 1d", "High")
Reduced-motion respected via tokens.css; bar charts render static
10 /

Hand-off & self-review

Engineering notes

  • App shell shared with F6/F9. The SideNav, top bar, and content frame come from @proxiserve/ui; the agent variant sets density="compact" + dark-mode capable + grouped nav. F9 admin uses the same shell with admin nav items.
  • Conversation lanes. One thread, three message types: client, agent, internal_note, plus system events. The API filters internal_note out of any client-facing read (F5/F6) at the query layer — not just the UI — so an internal note can never leak even via API.
  • Real-time. WebSocket channel per case for live message + status + assignment updates. Reassignment-away and admin-viewing banners are driven by the same channel. Offline writes queue and replay on reconnect.
  • SLA engine. Each service has a turnaround target; the queue computes age vs target and emits approaching (75%) and breached (100%) events. Breaches sort to top + trigger the banner + notify admin.
  • Document QC. Brightness/blur/face heuristics run server-side on upload (shared with F3); results cached and shown in the viewer. The agent's manual verdict is the authority — auto-checks only advise.
  • Assignment. Dispatcher assigns by service skill + current load + availability toggle + daily cap. Agents can pull from the unassigned pool when caught up.
  • Audit. Every status change, reassignment, document verdict, and admin view writes an immutable audit entry (surfaced in F9). Internal notes are retained but flagged staff-only.
  • Dark mode. Token swap on a data-theme="dark" root; persisted per user. Brand + status hues held constant for recognition.
  • Analytics. agent_today_view, agent_case_open {code}, agent_status_change {from,to}, agent_reply_sent, agent_note_added, agent_doc_verdict {accept|reject}, agent_reassign, agent_flag_admin, agent_palette_open, agent_unassigned_pull, agent_availability {on|off}.
