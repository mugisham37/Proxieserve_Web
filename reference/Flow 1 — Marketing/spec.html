Entry points

  • Direct URL — organic search, printed flyer, word of mouth.
  • Paid ads — Facebook / Google, often deep-linking to /services/[slug]; landing is the fallback.
  • WhatsApp share — link from an existing user.
  • Returning visitor — has localStorage.proxi:lastCode, banner offered.

Exit points

  • Start application → Flow 2 (Service Discovery) → Flow 3 (Submission).
  • Track an application → Flow 5 (Public Tracker, with code).
  • Sign in / Sign up → Flow 4 (Auth).
  • WhatsApp → external WhatsApp deep-link (pre-filled message includes referring page).

Accessibility checklist

Semantic heading outline — one h1 (hero), h2 per section, h3 for sub-blocks
Skip-to-content link present and focus-visible at top of every page
All interactive elements keyboard-reachable; visible :focus-visible ring (focus token)
Touch targets ≥ 44×44 px on mobile (pill buttons 48 px tall)
Contrast: body text 4.6:1 minimum measured; large heading 7.1:1
Colour never sole status carrier — pills carry icon + label
Form fields have real labels (visible) + helper + aria-describedby
Errors associated to inputs via aria-invalid + aria-describedby + role="alert"
Live regions: tracker submit status, cookie banner action, returning banner
Reduced motion: respected via single tokens.css media query — disables all transitions
Language attribute set on <html lang="en">; switcher updates DOM lang
All decorative images have aria-hidden="true"; content images have meaningful alt
Mobile sheet traps focus; Esc closes; returns focus to hamburger
Right-to-left mirror tested — application code stays LTR
Service cards: whole-card link, inner pill aria-hidden so SR doesn't double-announce
Language switcher: role="group" + native script labels (English, Kinyarwanda, Français)
FAQ accordion uses native <details>/<summary> — keyboard-native

Hand-off & self-review

Engineering notes

  • Stack assumption. Next.js 14 with App Router for SSR + static pages. Tailwind for utility class hints (tokens map directly).
  • Tokens. Exported as tokens.json; CSS custom properties + Tailwind config consume the same source.
  • Fonts. Self-host Fraunces + Inter + JetBrains Mono. Use font-display: swap. Preload only Fraunces 400 + Inter 400.
  • Images. Testimonial avatars are monogram-only by default; if real photos are added, serve AVIF + WebP with LQIP placeholder.
  • Analytics events. See section 1 list. All events route via a thin wrapper that respects user's cookie consent.
  • Service worker. Cache-first for landing shell + legal pages; stale-while-revalidate for services catalogue; network-only for tracker (sensitive).
  • SEO. Each route has unique <title>, meta description, OG image. Sitemap auto-generated. hreflang tags for EN/RW/FR.
  • Performance budget. LCP < 1.8 s on 3G fast; total JS < 80 kB gzipped on landing; CSS < 30 kB gzipped.
  • Image assets needed from client. Optional: real testimonial photos (3), team photos (4), office photo (1).

Designer self-review rubric (D.8)

01 — Real content (no lorem; Kinyarwanda annotation discipline kept in spec mode)
02 — Spacing scale obeyed (4/8/12/16/24/32/48/64/96/128)
03 — Semantic tokens only (hex only in tokens.css)
04 — Mobile-first (320 viewport renders without horizontal scroll)
05 — Sticky / safe-area (mobile CTA respects env(safe-area-inset-bottom))
06 — All states covered (12 in states.html)
07 — Reduced-motion variant (single tokens.css rule)
08 — A11y checklist signed (17/17 items)
09 — i18n length budget (no copy exceeds 80% of container at 1.35× expansion)
10 — Icon family consistent (Lucide line, 1.5 stroke; no emoji)
11 — Banned patterns absent (no purple, no isometric, no glass, no rainbow charts)
12 — Hierarchy obvious (eyebrow + h2 + section num scaffold consistent)
13 — CTAs unambiguous ("Start your application", "Track an application", "Send message", "WhatsApp")
14 — Edge cases (geo, archived code, multi-tab, draft expiry, schema mismatch — 12 documented)
15 — Final line invites next flow
