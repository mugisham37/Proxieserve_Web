Accessibility checklist

SideNav grouped with <nav> + section labels; active link aria-current
Charts have accessible data-table fallbacks (values exposed to SR, not image-only)
Schema builder fields keyboard-reorderable (not drag-only); inspector reachable by Tab
Field-type palette items are buttons; "add field" works without a pointer
Confirm-by-type input labelled; button aria-disabled until match, with text reason
Migration dialog focus-trapped; radio group labelled; Esc cancels
Audit log is a semantic table; kind tags carry text, not colour only
Status / capacity flags pair colour with text ("Over cap", "14 / 8")
Broadcast confirm announces reach + cost before send via live region
Dark mode meets AA contrast; brand/status hues constant across themes
Maintenance + delete actions need explicit typed confirmation (no accidental trigger)
Reduced-motion respected via tokens.css (bars + dialogs static)

Hand-off & self-review

Engineering notes

  • Schema is the source of truth. A service schema is a versioned JSON document. SchemaRenderer (shared with F3 step 2 + the builder's live preview) renders it identically everywhere. Publishing bumps version; the migration dialog writes a migrationStrategy that F3 applies on draft resume.
  • Effective-dated pricing. Prices are time-ranged rows. F7 validates the charge against the price effective at submission time, so a mid-flight price change never surprises a client (matches F7 state 10).
  • Audit log integrity. Append-only, hash-chained (each entry references the prior entry's hash). The API rejects any update/delete. No UI affordance to edit. 7-year retention; exportable for legal/compliance.
  • Privileged actions. Any admin action on a resource they don't own (view agent case, intervene, raise cap) requires a reason, notifies the affected party, and writes an audit entry tagged privileged.
  • Broadcasts. Segments are computed from live status, never manual lists. Opt-outs are subtracted at send time and are irreversible. Cost estimated per channel before send; test-to-self always available.
  • RBAC. Roles: agent, admin, owner. Some settings (maintenance mode, billing, deleting the last owner) are owner-only. Enforced server-side; the UI hides what the role can't do.
  • Analytics. Pre-aggregated rollups (hourly) for fast KPIs + on-demand recompute for custom ranges. Cached snapshot shown instantly with a refresh timestamp.
  • Dark mode. Admin default via platform setting; token swap on data-theme; per-user override persisted.
  • Analytics events. admin_view {page}, schema_publish {service,version,migration}, price_change {service,effective}, agent_cap_change, agent_invite, oversight_view {code}, admin_intervene {code,reason}, audit_filter, broadcast_send {segment,channels,reach}, setting_change {key}, maintenance_toggle.
