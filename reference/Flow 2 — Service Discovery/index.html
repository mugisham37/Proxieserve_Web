Key design decisions

Three calls worth flagging.

This is the page where every visitor decides whether to trust us. The decisions below are the ones I'd revisit first if any of them feel wrong.

01 / Card treatment

Coloured strip over coloured block

Flow 1's landing uses full coloured rectangles for service teasers (the candy moment). The full catalogue uses a quieter version — cream cards with an 8-12 px category-coloured strip at the top. Keeps density and scannability for 14+ services without overwhelming.

02 / Filtering

Single-select category, not multi

Civic-services users aren't shopping — they want one thing. Multi-category selection would invite indecision. Search + single-cat + sort is enough; the URL stays shareable and the IA stays explainable in one sentence.

03 / Detail layout

Action card sticks, never floats

The "Start application" CTA never disappears below the fold. Desktop: sticky right column. Mobile: bottom bar with fee + CTA. This is a conversion-first surface — the brief is to make starting feel one tap away.
