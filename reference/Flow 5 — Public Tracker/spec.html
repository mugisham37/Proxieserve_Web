Hero decisions worth flagging. The status hero is the entire emotional load of the page. It uses the notched motif (top-right + bottom-left notch — distinct from the marketing block direction). Color is status-driven, not decorative — magenta = in progress, info-blue = received, ok-green = completed, warn-yellow = action required / on hold, danger-red = rejected. The hero copy template is templated per status — agents don't write it.

Real-time updates

The tracker polls GET /api/track/[code] every 60 s while the tab is visible (Page Visibility API). Server returns a lastModified header; client only re-renders if the timestamp changed. WebSocket upgrade for the same endpoint is available — fallback to polling if connection fails.

Hand-off & self-review

Engineering notes

  • Code as bearer token. The PRX code is treated like a session token — short URL safe, log-rotated, and never sent in URL query strings (always path segment or POST body). 14 chars with checksum digit makes brute-force impractical.
  • Restricted view scope. Guest tracker shows status, timeline, agent identity, document list (names, dates), but never document previews — that requires authentication.
  • Status state machine. 9 statuses with allowed transitions; agents in F8 can only set the next valid status. The server enforces — admin can override with audit log entry.
  • SMS/WhatsApp parity. Every status change sends a templated SMS (always) and a WhatsApp template (if opted in). Both link to proxi.rw/t/PRX-…. Off-platform users get same info.
  • Polling. 60s interval while visible. Long-poll fallback if browser supports fetch keepalive. WebSocket upgrade gated by feature flag.
  • Caching. Tracker response has Cache-Control: private, max-age=30 + ETag. Reduces server load on shared codes during peak.
  • Rate limiting. Code entry endpoint: 5 attempts per IP per minute, then 1 per minute for 5 minutes. Per-code reads: unlimited (it's the rightful user's own check).
  • Analytics events. track_entry_view, track_code_submit {valid:bool}, track_view {code, status}, track_refresh {manual:bool}, track_agent_contact_click {channel}, track_doc_view_attempt, track_claim_modal_view, track_claim_modal_action {action}, track_pref_change {channel,on:bool}, track_lost_code_request.
