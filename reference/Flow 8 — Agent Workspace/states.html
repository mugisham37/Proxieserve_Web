Agent workspace — edge cases.

The agent surface is used all day, every day, often under time pressure. These states keep it fast, prevent costly mistakes, and degrade gracefully when work runs out or connections drop.

01 / Empty queue — all caught up

"You're all caught up"

Empty

All caught up.

No cases need you right now. New assignments will appear here. Enjoy the breather — or pull from the unassigned pool.

Reward, don't blank. An empty queue is a good thing for an agent — we frame it positively and offer the next useful action (pull unassigned work) rather than showing a sad empty box.

Stats first. The stat tiles + page header render instantly from cache; only the table skeletons. Skeleton appears after 300 ms so a fast load never flashes it.

04 / SLA about to breach

"2 cases need you now"

SLA
2 cases are over SLA. PRX-00483 (3d 4h) and PRX-00477 (2d 18h) have exceeded their turnaround target. These are pinned to the top of your queue.
Surfaced, not nagging. Over-SLA cases get a red age badge in the table, sort to the top, and trigger this banner once per session. Admin sees the same breaches in F9 oversight.
05 / Confirm before reassigning

"Reassign this case?"

Confirm

Reassign to another agent?

PRX-2026-00483 will leave your queue and go to the dispatcher for reassignment. The client won't be notified of the internal change. Add a handover note so the next agent has context.

Friction where it matters. Reassign, reject-document, and flag-for-admin all confirm first and capture a reason/note. Status changes (the common action) are one tap — only the rarer, costlier actions add friction.
06 / Internal note — never sent to client

Note mode is unmistakable

Safety
Only agents see this
The most important safety affordance in the flow. Internal-note mode turns the whole composer yellow, changes the placeholder, the label, and the button — visually impossible to mistake for a client reply. An agent should never accidentally send an internal note to a client.
07 / Offline — actions queue

"You're offline — changes saved locally"

Offline
You're offline. You can keep reviewing and writing — status changes, notes, and replies are saved on this device and sync the moment you reconnect. 3 CHANGES QUEUED
Agents work in spotty offices. The workspace is offline-tolerant: reads come from cache, writes queue locally with a visible count, and sync on reconnect. Sending a client message offline shows a "will send when online" pending state.

09 / Case taken / reassigned away

"This case moved"

Concurrency
This case was reassigned to Claudine N. by the dispatcher while you had it open. It's now read-only for you. Your draft reply was saved as an internal note so nothing is lost.
No silent data loss. If a case is reassigned or status-changed by someone else (admin, dispatcher) while you have it open, we go read-only and preserve any draft as a note. Real-time via the same channel pattern as F6's multi-tab handling.
10 / Admin viewing this case

"You're viewing as admin"

Oversight
Viewing as admin (read-only). This is Aline M.'s case. Actions are disabled — switch to oversight tools to reassign or intervene. Your view is logged.
Oversight without surveillance creep. When an admin opens an agent's case (from F9), this dark banner makes the elevated context explicit and the view is audit-logged. Admins can't silently act as an agent — every privileged view leaves a trail.
