06 /

PWA & offline

  • Manifest. Installable on Android + iOS (Add to Home Screen). Standalone display, cream theme-color, maskable icons. Install prompt deferred until after one successful interaction.
  • Service worker. Cache-first for the app shell + static assets; stale-while-revalidate for the services catalogue; network-only for payments + auth. Offline fallback page for uncached navigations.
  • Background Sync for queued writes (draft saves F3, messages F6/F8, document uploads). Web Push (VAPID) for notifications. IndexedDB for offline payloads.
  • Update flow. New SW version shows a subtle "Update available — refresh" toast rather than reloading under the user.
07 /

i18n & theming

  • Languages. EN (primary), RW (Kinyarwanda), FR — switchable everywhere, persisted per user, sent through to notification templates. Copy externalized to message catalogs; no hard-coded strings.
  • RTL-ready. Layout uses CSS logical properties throughout, so dir="rtl" mirrors the whole UI without a rebuild. Codes, phone numbers, and amounts forced LTR.
  • Theming. Light (public/client default) + dark (agent/admin default) via data-theme token swap. Brand + status hues constant across themes. Respects prefers-color-scheme first visit; per-user override persists.
  • Print. @media print sheet on every record surface (receipts, outcomes, summaries): chrome stripped, black-on-white, wordmark + legal footer, page-break-safe rows.
  • Motion. Global prefers-reduced-motion rule in tokens.css disables all transitions — inherited by all ten flows.

Hand-off & self-review

Engineering notes

  • Notification engine is a queue-backed service. Events publish to a topic; the engine resolves preferences, renders the per-channel template in the user's language, dispatches via channel adapters, and records delivery + read state. Idempotent per (event, user, channel).
  • Channel adapters: WhatsApp Business API, Africa's Talking SMS (Twilio failover), Web Push (VAPID), transactional email (SES/Postmark), in-app via WebSocket + persisted store.
  • Errors: a single error boundary renders the right page by status; 5xx auto-logs to Sentry with request context + incident ID. Maintenance is a feature flag flipped from F9.
  • PWA: Workbox-style SW with route-based caching strategies; Background Sync + Push; versioned cache with update toast.
  • i18n: message catalogs (ICU MessageFormat) for EN/RW/FR; the same catalog drives UI + notification templates. RTL via logical properties — no separate stylesheet.
  • Theming: CSS custom properties swapped on data-theme; SSR reads a cookie to avoid flash-of-wrong-theme.
  • Status page: separate static host + uptime monitor; not coupled to the main app's availability.
  • Analytics events. notif_sent {event,channel}, notif_delivered, notif_read, notif_pref_change, error_view {code}, pwa_install_prompt, pwa_installed, cookie_consent {choice}, theme_change, lang_change, onboarding_step {n}, onboarding_complete, status_view.
