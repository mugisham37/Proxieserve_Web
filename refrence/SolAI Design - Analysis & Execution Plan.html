SolAI Design System & Production Build Plan

A comprehensive analysis of the project documentation, design prompt, inspiration references, and 68 Stitch HTML files — distilled into a phased execution plan for building 13 flows (~70 screens) at production quality.

1. Inspiration Image Analysis

All 6 attached images come from RevenueBase — a B2B data infrastructure product. They establish a clear visual direction that aligns well with SolAI's "finance-grade, trustworthy" mandate.

Colour Temperature

Deep navy-blue background (~#0A1222) — not pure black. Surfaces use slightly lighter navy layers. This is cooler and more corporate than the Stitch files' #0a1421 but in the same family.

Typography & Rhythm

Mix of serif-italic for emotional headlines + clean sans-serif for body. Section labels use uppercase monospace tracking (e.g., "B2B DATA INFRASTRUCTURE", "PLATFORM"). Numbers are large and confident. The SolAI design tokens specify Inter + JetBrains Mono — we'll use JetBrains Mono for labels where RevenueBase uses uppercase spaced type.

Layout Patterns

  • Split hero — left text, right visual (layered diagrams)
  • Comparison tables — "Without / With" side-by-side with checkmarks/crosses, coloured bottom accent bars
  • Numbered FAQ accordions with gold accent underlines
  • Pricing columns with tabbed toggles (Data feeds / API credits)
  • Logo grid with "Testimonial" badges

Accent Strategy

Gold/amber as a warm accent against navy (testimonial badges, accent lines, bottom bars). Teal/green for success. Red/coral for negatives. Brand blue is restrained — used for links and CTA borders, not splashed everywhere. This is exactly the token set in DESIGN.md: --brand blue, --success green, --warning amber, --danger red.

Density & Spacing

Generous vertical spacing between sections. Text is given room to breathe. Cards use thin 1px borders on dark backgrounds, not shadows. Content sits within a centred max-width container (~1200px). Fine grid-dot backgrounds add subtle texture.

Trust Signals

Large stat callouts (e.g., "1B+ verification signals/month"). Numbered items. Comparison tables make confidence tangible. No flashy animations — calm authority. This maps perfectly to SolAI's "Why?" popover system and budget envelope bars.

Design Direction Summary

The final SolAI design will use: deep navy backgrounds (#0B0D12 per DESIGN.md), 1px bordered cards (no heavy shadows), uppercase monospace section labels, restrained blue accent with amber/gold as a warm secondary, generous vertical rhythm, and tabular numerals in JetBrains Mono for all financial data. The overall feel: a calm, finance-grade command centre — not a startup toy.

2. Stitch Reference Audit

The kl/ folder contains 68 HTML files covering all 13 flows. Each is a standalone Tailwind-based dark-mode page with Material Symbols icons.

What Works (Keep)

  • Comprehensive screen coverage — nearly every screen from the sitemap has a file
  • Colour base — #0a1421 navy is close to the locked #0B0D12; good starting point
  • Information architecture — the content grouping and nav structure is sound
  • Blueprint dot-grid background on dashboard — a nice subtle texture
  • Material Symbols icons are functional (we'll swap to Lucide per spec)

What's Weak (Fix)

  • No light mode — every file is dark-only; spec requires dark + light parity
  • No responsive design — all files are desktop-only; no mobile/tablet breakpoints
  • No state coverage — missing empty, loading, error, partial, permission states
  • No WhyPopover component — the spec's signature interaction is absent
  • No working interactions — static HTML, no toggles, no popovers, no tab switching
  • Inconsistent token usage — Material Design 3 colour system instead of locked DESIGN.md tokens
  • Material Symbols instead of Lucide — needs full icon swap
  • No accessibility — missing focus rings, ARIA labels, semantic HTML, contrast compliance
  • Lorem-style content — some screens use placeholder data instead of the required seed content
  • No consent banners, no multi-language rotation

4. Custom Components to Build

WhyPopover

The product's signature. A small "Why?" trigger on every auto-decided number. Opens a 240–360px popover with: headline, input bullets, agent + run ID, "Open full decision" link. Focus-trapped, Escape to close.

Critical Path

MoneyDisplay

Currency + amount + minor unit alignment. Tabular numerals. Optional "of cap" suffix. MoMo/Airtel tag. No fake .00 on zero-decimal currencies (RWF, KES).

Critical Path

AgentActivityRow

Live status row for each of the 5 agents. Avatar, name, current task, runtime, health pill, WhyPopover. States: running, idle, paused, errored.

Dashboard

BudgetEnvelopeBar

Three-tier nested bars: Swarm → Agent → Task. Spend vs cap with MoneyDisplay. Colour by cap state. WhyPopover on hover/focus.

Safety + Dashboard

ConsentBanner

Region-aware: GDPR (EU), Rwanda DPL (RW), POPIA (ZA). Proof export (JSON-LD). Never blocks page invisibly. Screen-reader announceable.

8. Key Design Decisions (Locked In)

1. Dark-first, RevenueBase-inspired

Deep navy #0B0D12 background. 1px bordered cards (no heavy shadows in dark). Uppercase mono section labels. Gold/amber as warm accent. Blue restrained to primary actions.

2. Sidebar App Shell

240px sidebar → 64px rail at 1024 → top-sheet drawer under 768. 56px topbar. All app screens (Flows 4–12) share this shell.

3. WhyPopover is First-Class

Every KPI, every auto-decision, every agent action gets a "Why?" trigger. It's not a tooltip — it's a popover with focus trap, keyboard support, and a link to the full audit trail.

4. Africa = Peer, Not Add-On

MoMo and Airtel sit next to Stripe in every payment flow. RWF and KES currencies are as prominent as USD. WhatsApp is the primary sales channel, not a secondary.

5. Lucide Icons Only

Replace all Material Symbols from Stitch with Lucide (1.5px stroke, 16/20/24px). No emoji in product UI.

6. Motion: Calm, Not Flashy

120–200ms transitions, ease cubic-bezier(.2,.8,.2,1). No spring physics. All animations honour prefers-reduced-motion.
