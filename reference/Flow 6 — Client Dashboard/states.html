Client dashboard — edge cases.

The authenticated home is where loyal users live. These ten cover the moments where it's empty, broken, contested, or asking for trust again.

01 / Empty — no applications yet

"Let's get started"

Pick a service and we'll help you apply on your behalf. Free to start — you only pay once we agree on the fee.

First-run. New account with no apps. Stats row hides; quick action chips stay. Banner above hero reads "Welcome to ProxiServe, Aline — your dashboard will populate as you apply."
02 / Loading — first paint

Skeleton shimmer

Loading
Order matters. Hero text is server-rendered, never blank. Skeletons only over data-dependent areas (stats, app cards, messages list). Reduced-motion: static blocks, no shimmer.
03 / Offline — cached snapshot

"You're offline"

Offline
You're offline. Showing what we saved last time — last sync 14:32.

Sending messages and uploading documents is disabled offline. The dashboard otherwise renders from cache; refresh button enabled.

PWA caching. Service worker caches dashboard shell + last response. New application button stays enabled but routes to "Connect to start an application" modal — same pattern as F1.
04 / Email verification banner

"Just one thing"

Pending verify
Confirm your email to unlock everything. We sent a link to aline.m@example.com — click it to verify. Without it, you can read but not start new applications.
Persistent until verified. Sits above the welcome banner across every dashboard page. Dismissible per-session but reappears next visit. Read-only mode applies (no new apps, no payments) until clicked.
05 / Message sending

Optimistic send

Pending
Got it, I'll retake the photo and upload now.

The reply appears optimistically — fades to fully opaque once delivered. If sending fails, a "Retry" link replaces the spinner.

Why optimistic. Half our users are on 3G — a 2-second sync delay before message appears feels broken. Pessimistic locks (waiting for server) are worse than retry on rare failure. Idempotency key handles double-sends.

07 / Multi-tab — same draft elsewhere

BroadcastChannel. Other tabs go read-only with a "your work was paused" banner — refreshing resumes. Last-writer-wins is unacceptable when settings are involved.
